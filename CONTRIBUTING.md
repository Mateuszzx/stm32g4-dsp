# Contributing to stm32g4-dsp

Thank you for your interest in contributing to the stm32g4-dsp project!

## How to Contribute

1.  **Fork the repository**.
2.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/my-new-feature
    ```
3.  **Make your changes**. Ensure your code follows the project's coding style.
4.  **Commit your changes** with descriptive commit messages.
5.  **Push to your branch**:
    ```bash
    git push origin feature/my-new-feature
    ```
6.  **Submit a Pull Request**.

## Reporting Issues

If you find a bug or have a suggestion, please open an issue in the repository. Provide as much detail as possible, including steps to reproduce the issue.

## Coding Standards

-   Follow the existing code style (indentation, naming conventions).
-   Ensure that the project builds successfully with CMake.
-   If modifying the DSP algorithms, please verify them using the Python utilities or appropriate tests.
